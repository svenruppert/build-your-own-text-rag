Purpose. Which Ollama model turns text into a vector. All stored vectors must come from the same model; changing this after adding entries makes the corpus inconsistent.

Values. Any embedding-capable model returned by Ollama. Defaults to nomic-embed-text-v2-moe when available.

Hint. The embedding dimension is fixed by the selected model and can differ between model families. Clear the store before switching, or added and query vectors will no longer compare meaningfully.
